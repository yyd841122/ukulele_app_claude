# 音频能力规划 (AUDIO_RECOGNITION_PLAN)

## 1. 音频能力分层

本项目音频能力分三个阶段实现，严格控制 MVP 阶段的技术风险和范围。

| 阶段 | 能力 | 实现方式 |
|------|------|----------|
| MVP | 调音器 + 录音 + 回放 + 手动自评 | 纯本地，无 AI |
| V1 | 单音音高评分 | 本地算法或云端 API |
| V2+ | 节奏评分、和弦识别、完整度评分 | 需 AI 模型 |

## 2. MVP 音频能力边界

### 2.1 调音器范围

| 项目 | MVP 规格 |
|------|----------|
| 支持乐器 | 21 寸尤克里里 |
| 调弦制式 | 标准 GCEA |
| 支持弦 | G / C / E / A 四根弦 |
| Low G | **不支持** |
| 多乐器 | **不支持** |
| 选择方式 | 手动选择弦 |
| 显示内容 | 当前频率（Hz）、最近音名、偏高/偏低指示 |
| 调音精度 | ±10  cents 范围内显示"准确" |
| 调音精度承诺 | 不承诺专业级精度 |
| 参考频率 | A4 = 440Hz |

**频率参考表**：

| 弦 | 音名 | 频率 (Hz) |
|----|------|-----------|
| 4 | G | 392.00 |
| 3 | C | 261.63 |
| 2 | E | 329.63 |
| 1 | A | 440.00 |

**偏高/偏低判断**：
- 偏高：检测频率 > 标准频率 + 10 cents
- 偏低：检测频率 < 标准频率 - 10 cents
- 准确：±10 cents 以内

### 2.2 录音范围

| 项目 | MVP 规格 |
|------|----------|
| 录音时长 | 最长 5 分钟 |
| 音频格式 | AAC (m4a) 或 WAV |
| 采样率 | 44100 Hz |
| 声道 | 单声道 |
| 存储位置 | App 本地私有目录 |
| 压缩 | 可选压缩，平衡质量与大小 |

### 2.3 回放范围

| 项目 | MVP 规格 |
|------|----------|
| 播放控制 | 播放 / 暂停 / 停止 |
| 进度显示 | 当前时间 / 总时长 |
| 进度拖拽 | 不支持 |
| 循环播放 | 不支持 |
| 倍速播放 | 不支持 |

### 2.4 自评范围

| 项目 | MVP 规格 |
|------|----------|
| 评分方式 | 手动三档选择 |
| 评分等级 | 好 / 一般 / 需改进 |
| AI 自动评分 | **不支持** |
| 音高自动检测 | **不支持** |
| 节奏自动检测 | **不支持** |

## 3. 技术验证项

### 3.1 调音器技术验证 (T009)

**验证目标**：确认 Flutter + Dart 能实现满足 MVP 需求的调音器。

**验证内容**：
1. 麦克风音频采集延迟 < 50ms
2. 频率检测算法精度 ±5 cents
3. UI 刷新率 30fps
4. CPU 占用 < 30%

**验证方案**：
- 使用 record 包获取实时音频流
- Dart 实现 FFT（推荐 use fft 包）或自相关算法
- 不依赖 native 插件（简化 MVP）

**通过标准**：
- 能稳定检测 G / C / E / A 四个音
- 能在 2 秒内给出检测结果
- 无明显卡顿或崩溃

**风险**：
- 如果纯 Dart 方案精度/性能不足，再考虑使用 native ftt 插件
- 备选：flutter_sound 或 pitch_detector 包

### 3.2 录音回放技术验证 (T011)

**验证目标**：确认 record + just_audio 能满足录音回放需求。

**验证内容**：
1. 录音启动延迟 < 200ms
2. 回放启动延迟 < 200ms
3. 录音文件完整可播放
4. 权限申请正常

**验证方案**：
- 使用 record 进行录音
- 使用 just_audio 进行回放
- 权限使用 permission_handler

**通过标准**：
- 能录制 5 分钟音频
- 能正常回放录音
- 文件能保存到本地

## 4. 后期音频能力路线

### 4.1 V1: 音高评分

**目标**：对单音练习进行音高准确度评分。

**技术方案**（待验证）：
- 本地方案：使用 fft + 音高检测算法
- 云端方案：调用音频分析 API

**接口预留**：

```dart
abstract class PitchEvaluationService {
  Future<PitchResult> evaluate({
    required String audioPath,
    required String expectedNote, // e.g., 'C', 'G'
  });
}

class PitchResult {
  final int score; // 0-100
  final double detectedFrequency;
  final double expectedFrequency;
  final double deviationCents;
  final String feedback;
}
```

### 4.2 V2: 节奏评分

**目标**：评估节拍稳定性。

**技术方案**（待验证）：
- 检测节拍时间点
- 计算与设定 BPM 的偏差
- 生成节奏稳定性报告

### 4.3 V2: 和弦识别

**目标**：识别用户弹奏的和弦。

**技术方案**（待验证）：
- 音频特征提取
- 和弦模板匹配
- 和弦转换时间检测

### 4.4 V2: 扫弦方向识别

**目标**：识别下扫/上扫/休止。

**接口预留**：

```dart
abstract class StrummingRecognitionService {
  Future<StrummingPattern> recognize(String audioPath);
}

class StrummingPattern {
  final List<StrumDirection> directions; // down, up, rest
  final double tempo;
}
```

### 4.5 V3: 演奏完整度评分

**目标**：综合评估演奏效果。

**技术方案**（待验证）：
- 音高准确度
- 节奏准确度
- 和弦转换流畅度
- 整体完成度

## 5. 技术债务与风险

| 风险 | 级别 | 缓解措施 |
|------|------|----------|
| 调音器精度不足 | 中 | MVP 不承诺专业精度 |
| 音频延迟影响体验 | 中 | 技术验证先行 |
| AI 评分方案不确定 | 高 | MVP 不做 AI，预留接口 |
| 跨平台音频差异 | 低 | Android 优先测试 |

## 6. 禁止事项

以下在 MVP 阶段**明确禁止**：

- ❌ 联网 AI API 调用
- ❌ 实时音高校准（如专业级调音）
- ❌ 自动和弦识别
- ❌ 自动扫弦方向识别
- ❌ 演奏完整度 AI 评分
- ❌ 录音上传云端分析
