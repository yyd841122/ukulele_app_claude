# 合规文档 (COMPLIANCE)

> 本文档定义 ukulele_app 在隐私、安全、版权等合规层面的要求和注意事项。

## 1. 麦克风权限

### 1.1 权限声明

| 平台 | 权限 | 用途 |
|------|------|------|
| Android | RECORD_AUDIO | 调音器、录音 |
| Android | READ_EXTERNAL_STORAGE | 读取录音文件 |
| Android | WRITE_EXTERNAL_STORAGE | 保存录音文件 |
| iOS | NSMicrophoneUsageDescription | 调音器、录音 |

### 1.2 权限申请时机

| 场景 | 申请时机 |
|------|----------|
| 调音器 | 用户首次点击"开始调音"时 |
| 录音 | 用户首次点击"录音"时 |
| 回放 | 不需要权限 |

### 1.3 权限拒绝处理

- 用户拒绝权限：显示友好提示，说明为何需要该权限
- 提供"重试"按钮
- 不强迫用户授权
- 降级处理：调音器可降级为手动调音参考表，录音不可用

## 2. 用户录音隐私

### 2.1 录音存储

| 项目 | 说明 |
|------|------|
| 存储位置 | App 私有目录（getApplicationDocumentsDirectory） |
| 访问权限 | 仅 App 自身可访问 |
| 备份 | 默认不自动备份到云端 |
| 导出 | 不提供录音导出功能（MVP） |

### 2.2 录音数据归属

- 用户录音归用户个人所有
- App 不收集、不上传、不分享用户录音
- 用户卸载 App 后，录音数据一并删除
- **MVP 不提供录音导出**，防止版权问题

### 2.3 录音文件格式

- 使用 AAC (m4a) 格式，平衡质量与大小
- 或使用 WAV 格式保证质量
- 文件名使用时间戳或 UUID，不包含敏感信息

## 3. 本地优先策略

### 3.1 数据不出境

| 数据类型 | 存储位置 |
|----------|----------|
| 练习记录 | 本地 SQLite |
| 用户设置 | 本地 SharedPreferences |
| 录音文件 | App 私有目录 |
| 用户内容 | App 私有目录 |

### 3.2 MVP 无网络依赖

- App 在飞行模式下完整功能可用
- 不检测网络状态
- 不显示网络错误

### 3.3 后期云同步风险预留

未来如需云同步，必须：

1. 明确用户同意数据上传
2. 提供数据删除机制
3. 符合 GDPR（若面向欧盟用户）
4. 符合各平台隐私政策

## 4. 未成年人风险预留

### 4.1 年龄分级

- Google Play: 早期教育类别需特别注意儿童隐私
- App Store: 需明确年龄分级

### 4.2 COPPA 合规预留

如未来面向 13 岁以下用户：
- 不收集个人信息
- 不定向广告
- 需父母同意机制

### 4.3 MVP 说明

MVP 主要面向项目 owner 本人使用，暂不涉及未成年人保护。但后期上架时需重新评估。

## 5. AI 生成内容风险预留

### 5.1 AI 评分风险

| 风险 | 缓解措施 |
|------|----------|
| 评分不准确导致用户误导 | MVP 不做 AI 评分 |
| AI 服务中断 | MVP 本地算法 |
| AI 偏见 | 评分仅作参考，不作唯一标准 |

### 5.2 AI 曲谱生成风险

未来如引入 AI 曲谱生成：
- 明确告知用户内容为 AI 生成
- 不生成特定艺术家风格的歌曲
- 避免版权争议

## 6. 歌曲版权风险

### 6.1 MVP 版权策略

- **不内置**任何未经授权的商业歌曲完整歌词
- **不内置**任何未经授权的商业曲谱
- **不内置**任何未经授权的完整歌曲音频

### 6.2 可内置内容

| 内容类型 | 说明 |
|----------|------|
| 公版旋律 | 版权到期或明确公版的曲目 |
| 自制练习曲 | 项目 owner 原创 |
| 基础练习片段 | 简单和弦进行、音阶练习 |
| 公版和弦进行 | 如 I-IV-V-I 等通用进行 |

### 6.3 版权到期时间参考

| 地区 | 版权规则 |
|------|----------|
| 中国大陆 | 作者死后 50 年 |
| 欧盟 | 作者死后 70 年 |
| 美国 | 1978 年前作品 95 年；1978 年后作者死后 70 年 |

### 6.4 用户生成内容 (UGC)

| 场景 | 策略 |
|------|------|
| 用户输入歌词 | 仅本地存储，不上传、不分享 |
| 用户输入曲谱 | 仅本地存储，不上传、不分享 |
| 用户录音 | 归用户所有，可自行导出（后期功能） |

## 8. 安全建议

### 8.1 本地数据安全

| 建议 | 实现 |
|------|------|
| 敏感数据加密 | MVP 暂不需要，录音文件本身非敏感 |
| 安全存储 | 使用 App 私有目录 |
| SQL 注入 | Drift ORM 自动防护 |

### 8.2 未来云同步安全

如后期实现云同步：
- HTTPS 传输
- JWT 或 OAuth 2.0 认证
- 数据加密存储
- 定期安全审计
