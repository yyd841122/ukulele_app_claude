# 内容策略 (CONTENT_POLICY)

> 本文档定义 ukulele_app 的内容边界、版权合规要求和用户生成内容管理策略。

## 1. 内容原则

**核心原则**：MVP 阶段不碰商业歌曲版权，只提供自制内容和公版内容。

## 2. MVP 内置内容策略

### 2.1 可内置内容

以下类型内容可以在 MVP 中安全内置：

| 内容类型 | 示例 | 版权说明 |
|----------|------|----------|
| 尤克里里音阶练习 | C 大调音阶、G 大调音阶 | 音乐音阶无版权 |
| 基础和弦练习 | C-G-Am-F 经典进行 | 和弦本身无版权 |
| 简单练习曲 | 自制短旋律 | 原创，无版权 |
| 公版旋律 | 《小星星》旋律 | 公版（原作者佚名或已到期） |
| 基础节奏型 | 4/4 拍全音符到十六分音符 | 节奏型无版权 |
| 调音参考 | G=392Hz, C=261.63Hz 等 | 音乐理论，无版权 |

### 2.2 明确禁止内置

以下内容**严格禁止**在 MVP 中内置：

| 内容类型 | 禁止原因 |
|----------|----------|
| 商业歌曲完整歌词 | 歌词版权归属词作者 |
| 商业歌曲完整曲谱 | 曲谱版权归属曲作者/出版商 |
| 商业歌曲和弦谱 | 和弦编配可能有版权 |
| 未经授权的翻唱音频 | 音频版权归属唱片公司 |
| 知名艺术家作品 | 全部有版权保护 |
| 近期热门歌曲 | 版权活跃期 |

### 2.3 版权风险判断流程

```
内容是否内置？
├── 是否是原创自制内容？ → YES → 可以内置
├── 是否是公版内容？ → YES → 可以内置
├── 是否明确无版权（如音阶、基础节奏型）？ → YES → 可以内置
└── 其他 → NO → 不能内置
```

## 3. 公版内容参考

### 3.1 公版判断标准

符合以下任一条件即为公版：
- 作者去世超过 50 年（中国大陆）/ 70 年（欧盟/美国）
- 明确声明放弃版权
- 属于公共领域（如音乐理论、和弦进行）

### 3.2 常见公版曲目参考

| 曲目 | 状态 | 注意事项 |
|------|------|----------|
| 《小星星》(Twinkle Twinkle) | 公版（旋律古老） | 可能仍有编曲版本受版权 |
| 《生日快乐》 | 公共领域 | 旋律古老，公版 |
| 简单儿歌 | 多为公版或自制 | 需逐一确认 |

**谨慎原则**：即使是公版曲目，如果使用特定编曲版本可能仍有版权。MVP 建议只使用原创或明确无争议的内容。

## 4. 用户生成内容 (UGC) 管理

### 4.1 UGC 范围

| UGC 类型 | MVP 处理 |
|----------|----------|
| 用户输入歌词 | 仅本地存储，不上传、不分享 |
| 用户输入曲谱 | 仅本地存储，不上传、不分享 |
| 用户录音 | 归用户所有，可本地回放 |
| 用户创建的练习计划 | 仅本地存储 |

### 4.2 UGC 存储策略

```
用户创建内容
  → 存储在 App 私有目录
  → 不上传到任何服务器
  → 不与任何第三方共享
  → 卸载 App 后数据删除
```

### 4.3 UGC 版权声明

在设置页或相关页面显示：

> "您在本应用中创建的内容（录音、笔记等）归您个人所有。App 不会上传、分享或用于任何其他目的。"

## 5. 后期商业化内容策略

### 5.1 歌曲库商业化路径

| 路径 | 说明 |
|------|------|
| 购买版权 | 与版权方直接授权 |
| 版权代理 | 通过音著协等代理机构 |
| UGC 合规 | 用户自行上传，自行承担版权责任 |
| 自制内容 | 继续自制练习曲 |

### 5.2 内容合作建议

| 合作方 | 用途 |
|------|------|
| 音乐出版社 | 获取曲谱授权 |
| 唱片公司 | 获取音频/歌词授权 |
| 独立音乐人 | 合作原创内容 |
| 音著协 / ASCAP / BMI 等 | 版权代理服务 |

## 6. 免责声明要求

### 6.1 App 内免责声明

在相关功能页面显示：

> "本应用内置内容均为原创或公版内容。我们尊重知识产权，不提供未经授权的商业歌曲内容。"

### 6.2 用户协议中的版权条款

后期编写用户协议时需包含：

- 内容版权归属说明
- UGC 版权责任声明
- 侵权投诉联系方式
- 版权到期内容下架机制

## 7. 违规内容处理

### 7.1 举报机制

如发现 App 中有侵权内容，提供举报通道：
- 设置页"意见反馈"入口
- 举报邮箱（后期）

### 7.2 处理流程

```
收到举报
  → 核实内容版权状态
  → 如确认侵权，移除内容
  → 记录并反思内容引入流程
```

## 8. 定期审查

| 审查时间 | 审查内容 |
|----------|----------|
| MVP 发布前 | 确认所有内置内容无版权风险 |
| V3 发布前 | 审查新增歌曲库内容 |
| 定期（如每季度） | 审查是否有新增侵权内容 |

## 9. 内容团队协作

### 9.1 内容引入流程

```
Music Domain Expert 提出内容需求
  → Compliance Reviewer 审查版权
  → Chief Architect 审批
  → 才能添加到代码库
```

### 9.2 禁止事项

- ❌ Agent 擅自添加非自制/非公版内容
- ❌ 使用"应该没问题"侥幸心理
- ❌ 先上线后补充版权
