# 产品需求文档 (PRD) - MVP 草稿

> 本文档为 MVP 阶段产品需求初稿，将随着项目推进逐步细化。

## 1. 产品背景

### 1.1 市场现状

尤克里里（Ukulele）作为一种便携、易入门的弦乐器，在全球拥有大量爱好者。但市面上的练习应用普遍存在以下问题：

- 大部分应用面向吉他，对尤克里里针对性弱
- 练习内容分散，用户不知道每天该练什么
- 缺乏有效的自我评估手段，练习效果不可见
- 完整歌曲练习门槛过高，初学者容易放弃

### 1.2 目标机会

针对 21 寸尤克里里标准 GCEA 调弦，提供一套从单音到弹唱的渐进式练习体系，让用户每天都有明确的练习目标，并能通过录音回放进行自我评估。

## 2. 目标用户

### 2.1 主要用户

| 用户画像 | 描述 |
|----------|------|
| 核心用户 | 尤克里里初学者，零基础或刚学会基本持琴姿势 |
| 使用场景 | 每天在家练习 15-30 分钟 |
| 核心诉求 | 知道练什么、知道练得怎么样 |
| 平台偏好 | Android 手机，碎片化时间练习 |

### 2.2 用户故事

**Story 1: 每日练习迷茫**
> 作为一个初学者，我打开 App 就能看到"今天该练什么"，不需要自己规划练习内容。

**Story 2: 调音需求**
> 每次练习前，我需要确认琴弦音准，App 内置调音器比外部 App 更方便。

**Story 3: 节奏感培养**
> 我刚开始打拍子总是不稳，节拍器能帮我建立稳定的节奏感。

**Story 4: 录音自评**
> 我弹了一段后想听听效果，录音回放能帮我发现自己的问题。

**Story 5: 进度可见**
> 我练了一周想知道有没有进步，练习记录能展示我的坚持和变化。

## 3. 核心痛点

| 痛点 | 描述 | MVP 解决方案 |
|------|------|--------------|
| 不知道每天练什么 | 初学者缺乏练习方向 | 今日练习计划 |
| 不清楚自己练得怎么样 | 缺乏自我评估手段 | 录音 + 回放 |
| 练习效果不可见 | 不知道坚持练习有没有用 | 本地练习记录 |
| 每次要找外部工具 | 调音器、节拍器要开别的 App | 内置调音器、节拍器 |

## 4. MVP 目标

### 4.1 产品目标

构建一款**本地离线优先**的尤克里里练习应用，提供清晰的每日练习路径，让用户坚持练习并看到进步。

### 4.2 成功指标（定性，非定量）

- 用户能独立完成一次完整的练习流程：打开 App → 看今日计划 → 调音 → 练习单音/和弦 → 用节拍器 → 录音自评 → 保存记录
- 用户能在没有网络的环境下完成上述全部流程
- MVP 阶段不设 DAU/留存等商业指标，以功能完整性和用户体验为主要验收标准

## 5. 功能范围

### 5.1 MVP 必做功能

| 功能模块 | 功能点 | 优先级 |
|----------|--------|--------|
| 首页 | 今日练习计划展示 | P0 |
| 调音器 | GCEA 标准调弦，麦克风检测频率 | P0 |
| 单音练习 | C / F / Am / G 等音名按弦练习 | P0 |
| 和弦库 | 基础和弦列表与指法图 | P0 |
| 节拍器 | 可调 BPM，基础节奏型 | P0 |
| 录音 | 麦克风录制练习片段 | P0 |
| 回放 | 播放已录制的录音 | P0 |
| 自评 | 手动自评（好/一般/需改进） | P0 |
| 练习记录 | 本地保存每日练习记录 | P0 |
| 设置 | 基础设置项 | P0 |

### 5.2 MVP 不做功能

以下功能**明确不做**，任何需求扩展必须经过 Chief Architect 和 Product Manager 双重确认：

- 账号登录 / 注册
- 云端数据同步
- AI 自动评分
- 完整歌曲库
- 社区 / 排行榜
- 内购 / 订阅
- iOS 版本（MVP 仅 Android）
- Low G 调弦
- 多乐器支持

## 6. 非功能需求

### 6.1 性能需求

- App 冷启动时间 < 3 秒
- 调音器响应延迟 < 100ms
- 录音到回放的延迟 < 200ms
- 节拍器精度误差 < 5 BPM（在可接受范围内）

### 6.2 兼容性需求

- 最低支持 Android SDK 21 (Android 5.0)
- 推荐 targetSdk 34
- MVP 不支持平板 UI 优化
- MVP 不支持深色模式（后续版本预留）

### 6.3 隐私需求

- 所有用户数据仅保存在本地
- 麦克风权限仅在录音/调音时请求
- 不收集任何用户行为数据
- 不包含任何追踪器或埋点

## 7. 内容合规

- MVP 不内置任何未经授权的商业歌曲完整歌词
- MVP 不内置任何未经授权的商业曲谱
- MVP 内置内容：自制练习曲、公版旋律、基础和弦练习片段
- 用户输入内容仅作为个人练习用途，不公开传播

详细合规策略见 [CONTENT_POLICY.md](CONTENT_POLICY.md)

## 8. 技术约束

- Flutter + Dart
- Android first，iOS 预留
- 本地 SQLite 存储（Drift）
- 无后端服务
- 无 AI/API 集成

详细技术选型见 [TECH_STACK.md](TECH_STACK.md)

## 9. MVP 验收标准

### 9.1 功能验收

| 功能 | 验收条件 |
|------|----------|
| 今日练习 | 能看到当天练习任务列表 |
| 调音器 | 能检测 G/C/E/A 四弦的音准状态 |
| 单音练习 | 能播放音名并显示按弦指引 |
| 和弦库 | 能浏览和弦列表并查看指法图 |
| 节拍器 | 能按设定 BPM 发声 |
| 录音 | 能录制最长 5 分钟音频 |
| 回放 | 能播放已录制音频 |
| 自评 | 能对录音进行自评并保存 |
| 练习记录 | 能查看历史练习日期和内容 |
| 设置 | 能修改基础设置项 |

### 9.2 体验验收

- [ ] 所有功能在飞行模式下正常工作
- [ ] 首次使用能顺利完成调音 → 练习 → 录音 → 保存全流程
- [ ] 无崩溃、无明显卡顿

## 10. 后期演进

### 10.1 V1 演进方向

- 单音音高准确度评分
- 基础节奏准确度反馈
- 练习计划个性化

### 10.2 V2 演进方向

- 和弦转换流畅度评估
- 扫弦方向识别
- 演奏完整度评分

### 10.3 V3 演进方向

- 整曲练习模式
- 完整歌曲和弦歌词谱
- 多调弦支持

### 10.4 V4/V5 演进方向

- AI 曲谱生成
- 账号系统
- 云同步
- 商业化上架

> 每次演进都必须重新评估合规风险，特别是涉及用户生成内容、云同步、AI 评分等功能时。
