## 1. 产品背景

### 1.1 市场现状（基于通用产品规律，**未外部复核**）

尤克里里（Ukulele）是一种便携、易入门的弦乐器，在全球拥有大量爱好者。但市面上的练习类应用普遍存在以下问题（基于 T001 初步研究 + 通用产品规律）：

- 大部分应用面向吉他，对尤克里里针对性弱；
- 练习内容分散，用户不知道每天该练什么；
- 缺乏有效的自我评估手段，练习效果不可见；
- 完整歌曲练习门槛过高，初学者容易放弃；
- 内置商业歌曲存在版权风险。

### 1.2 项目机会

针对 **21 寸尤克里里标准 GCEA 调弦**，提供一套从单音到弹唱的渐进式本地化练习体系，让用户每天都有明确的练习目标，并能通过录音回放进行自我评估。

### 1.3 研究背景与不确定性声明

> **本节是后续所有 Agent 解读本文档的边界声明，请勿忽视。**

## 2. 目标用户

### 2.1 核心用户画像

| 维度 | 描述 |
|------|------|
| 身份 | 尤克里里初学者（零基础或刚学会基本持琴姿势） |
| 设备 | Android 手机（中端及以上，Android 6.0+） |
| 场景 | 每天在家练习 15-20 分钟 |
| 核心诉求 | 知道练什么、知道练得怎么样 |
| 网络条件 | 可能无网络（地铁、户外） |

### 2.2 MVP 不面向的用户

- 专业演奏者（MVP 不承诺专业精度）；
- 13 岁以下儿童（MVP 不做 COPPA 专项合规）；
- iOS / iPad 用户（MVP 不支持）；
- 多乐器用户（MVP 仅尤克里里）；
- Android 6.0 以下设备用户（minSdk 23 限制）；
- 鸿蒙 / Android Go / 特殊定制 ROM 用户（MVP 不专门适配）。

## 3. 核心痛点

| 痛点 | 描述 | MVP 解决方案 |
|------|------|--------------|
| 不知道每天练什么 | 初学者缺乏练习方向 | 预设 7 天循环练习计划 |
| 不清楚自己弹得怎么样 | 缺乏自我评估手段 | 录音 + 回放 + 三档手动自评 |
| 练习效果不可见 | 不知道坚持有没有用 | 本地练习记录 + 历史列表 |
| 每次要找外部工具 | 调音器、节拍器要开别的 App | 内置调音器、节拍器 |
| 完整歌曲门槛太高 | 曲库版权风险 + 难度高 | 自制练习片段 + 公版内容 |

## 4. 产品目标

### 4.1 MVP 目标

构建一款 **Android 平台、本地离线优先** 的尤克里里练习应用，提供清晰的每日练习路径，让初学者坚持练习并看到进步。

### 4.2 成功标准（定性）

- 用户能独立完成一次完整的练习流程：打开 App → 看今日计划 → 调音 → 练习单音/和弦 → 用节拍器 → 录音自评 → 保存记录；
- 用户能在飞行模式下完成上述全部流程；
- 单次练习时长控制在 15-20 分钟内可完成；
- MVP 阶段不设 DAU/留存等商业指标，以功能完整性和用户体验为主要验收标准。

---

## 5. MVP 核心闭环

MVP 的核心业务闭环为：

```text
打开 App
  → 看到今日练习（预设 Day N 任务列表）
  → 练习前可调音（G/C/E/A 手动选弦）
  → 进入单音 / 和弦 / 节奏小练习
  → 可使用节拍器（基础 4/4，50-200 BPM）
  → 可录音（最长 5 分钟，本地保存）
  → 可回放（播放 / 暂停 / 停止）
  → 手动自评（好 / 一般 / 需改进）
  → 保存本地练习记录（SQLite）
  → 下次打开能看到历史进度
```

闭环中每一步都必须是**可独立使用**的功能模块，无强制依赖。

## 6. MVP 功能范围

### 6.1 首页 / 今日练习

| 项目 | 规格 |
|------|------|
| 入口 | App 启动后默认页 |
| 内容 | 显示当天练习任务卡片列表 |
| 数据源 | 预设 7 天循环（Day 1-7） |
| 任务项 | 单音 / 和弦 / 节拍 / 录音 / 自评 任务组合 |
| 状态 | 标记完成 / 未完成，不强制全部完成 |
| 算法 | **无 AI 推荐，无个性化**，纯按天序号轮换 |
| 轮换逻辑 | `Day N = (自安装起的天数 % 7) + 1`，Day 1-7 严格循环 |

### 6.2 基础调音器

| 项目 | 规格 |
|------|------|
| 支持乐器 | 21 寸尤克里里 |
| 调弦制式 | **标准 GCEA**（G4/C4/E4/A4） |
| 支持弦 | G / C / E / A 四根弦，**手动选择** |
| Low G | **不支持** |
| 多乐器 | **不支持** |
| 显示 | 当前频率（Hz）、目标频率、偏高/偏低/接近准确提示 |
| 参考频率 | A4 = 440Hz |
| 频率参考表 | G=392.00 / C=261.63 / E=329.63 / A=440.00 |
| 精度 | ±10 cents 是**体验目标**，**非硬验收** |
| 降级 | 如实时检测不稳定，UI 标记"实验性调音器"，**同屏**保留 GCEA 频率表 + 手动调音说明 |
| 精度承诺 | **不承诺专业级调音精度** |
| 自动选弦 | **不做** |

**降级 UI 形态**（同屏示例）：
```text
┌──────────────────────────┐
│  实验性调音器 ⚠          │
├──────────────────────────┤
│ 当前检测频率：   390.2 Hz │
│ 目标频率（G4）： 392.0 Hz │
│ 偏差：            -8 cents│
│ 状态：接近准确           │
├──────────────────────────┤
│  GCEA 标准频率表         │
│  G4 = 392.00 Hz          │
│  C4 = 261.63 Hz          │
│  E4 = 329.63 Hz          │
│  A4 = 440.00 Hz          │
├──────────────────────────┤
│  [G弦] [C弦] [E弦] [A弦] │
└──────────────────────────┘
```

### 6.3 单音练习

| 项目 | 规格 |
|------|------|
| 范围 | 基础音名按弦练习（C / D / E / F / G / A / B） |
| 内容展示 | 音名 + 弦号 + 品格位置（静态文字 + 自绘图） |
| 声音引导 | MVP **不内置音色播放**（避免音频素材版权问题），仅展示指位 |
| AI 评分 | **不做** |

### 6.4 基础和弦库与指法图

| 项目 | 规格 |
|------|------|
| 和弦范围 | **C / Am / F / G / G7 / Dm / Em**（共 7 个） |
| 展示方式 | 自绘静态指法图（PNG / SVG 资源） |
| 编排 | ChordLibraryPage → ChordDetailPage |
| 动画 | **不做交互式动画** |
| TAB / 五线谱 / 简谱 | **不做** |
| 商业和弦谱 | **不做** |

### 6.5 节拍器

| 项目 | 规格 |
|------|------|
| 拍号 | **仅基础 4/4 拍** |
| BPM 范围 | **50 - 200** |
| 默认 BPM | **80**（适合初学者） |
| 控制 | 开始 / 暂停 / 停止 |
| 调节 | BPM 滑块或加减按钮 |
| 节奏型 | **不做**（仅 4/4 全拍） |
| 鼓机 | **不做** |
| 后台播放 | **不做** |
| 声音 | 自制 / CC0 节拍器音 |

### 6.6 录音

| 项目 | 规格 |
|------|------|
| 最长时长 | **5 分钟**（硬上限） |
| 接近上限提示 | 录音到 **4:30** 时 UI 提示"还剩 30 秒"，到 5 分钟自动停止 |
| 格式 | AAC (m4a)，44.1kHz，单声道 |
| 存储位置 | App 私有目录（`getApplicationDocumentsDirectory()`） |
| 上传 | **不上传** |
| 导出 | **不导出** |
| 分享 | **不分享** |
| 删除 | 用户可在历史记录中删除本地录音 |
| 每次练习关联 | 默认最多关联**一段**录音 |
| 蓝牙耳机 | 可选支持（`MODIFY_AUDIO_SETTINGS`） |
| 后台录音 | **不做** |

### 6.7 回放

| 项目 | 规格 |
|------|------|
| 控制 | 播放 / 暂停 / 停止 |
| 进度显示 | 当前时间 / 总时长 |
| 进度拖拽 | **不支持** |
| 循环播放 | **不支持** |
| 倍速 | **不支持** |
| 录音文件丢失处理 | 若录音文件已被外部删除，UI **隐藏回放按钮**，不显示错误 |

### 6.8 手动自评

| 项目 | 规格 |
|------|------|
| 方式 | **手动三档选择** |
| 等级 | 好 / 一般 / 需改进 |
| AI 自动评分 | **不支持** |
| 音高自动检测（演奏场景） | **不支持** |
| 节奏自动检测 | **不支持** |
| 关联 | 自评必须可关联练习记录（一条练习记录 + 一个自评） |
| 事后修改 | **不支持**事后修改（避免引入"自评数据被反复修改"的复杂度） |

### 6.9 本地练习记录

| 项目 | 规格 |
|------|------|
| 存储 | 本地 SQLite（Drift） |
| 记录字段 | 见 §10.1 |
| 列表 | 按日期倒序 |
| 删除 | 支持单条删除；**级联删除**关联录音文件 |
| 删除录音文件 | UI 检测到录音文件不存在时**隐藏回放按钮**，记录本身保留 |
| 导出 | MVP **不导出** |
| 同步 | MVP **不同步**，无云端 |

### 6.10 设置 / 关于 / 隐私说明

| 项目 | 规格 |
|------|------|
| 设置项 | 节拍器默认 BPM（默认 80）、节拍器音量 |
| 关于页 | 版本号、开源许可 |
| 隐私说明页 | 数据收集声明、麦克风权限用途、录音本地存储说明 |
| 内容声明页 | 内置内容版权说明 |
| 反馈 | MVP **不内置反馈通道**（后期加） |
| 账号 | MVP **无账号体系** |
| 深色模式 | MVP **不做深色 UI**，但 ThemeData 设计预留切换能力 |

## 7. MVP 不做什么

以下功能在 MVP 阶段**明确禁止**，任何 Agent 不得擅自添加：

### 7.1 账号与网络

- 账号注册 / 登录；
- 云端数据同步；
- 第三方登录；
- 任何联网 API 调用；
- 推送通知；
- 任何第三方分析 SDK（如 Firebase / Sentry / Bugly）。

### 7.2 AI 与评分

- AI 自动音高评分；
- AI 节奏分析；
- AI 和弦识别；
- AI 自动扒谱；
- 任何调用外部 AI 服务的能力。

**重要澄清**：调音器的基础频率检测（G/C/E/A 单音 + 偏高/偏低提示）**不属于** AI 评分，是调音器所需的最小技术能力。两者的技术路径完全不同：
- 调音器：单帧/短窗音频 → 数字频率 → 与目标频率比较 → 方向提示；
- AI 评分：多帧音频 → 模型推理 → 综合评分/反馈。

### 7.3 社区与商业

- 用户社区；
- 排行榜；
- 好友系统；
- 分享功能；
- 内购 / 订阅；
- 广告。

### 7.4 内容版权

- 商业歌曲完整歌词；
- 未经授权的商业曲谱；
- 完整歌曲库；
- 用户输入歌曲名生成曲谱；
- 用户输入歌词；
- 用户输入 YouTube / B 站 / 网易云链接；
- AI 曲谱生成。

### 7.5 平台与设备

- iOS / iPad 版本；
- Web / PWA；
- 桌面端；
- 多乐器支持；
- Low G 调弦；
- TAB / 五线谱 / 简谱 / 混合谱；
- 平板 UI 优化；
- 深色模式 UI；
- 鸿蒙 / Android Go 专项适配。

### 7.6 调音器扩展

- 专业级调音精度承诺；
- 自动选弦；
- 多弦同时检测；
- 商业级调音器宣传文案。

## 8. 用户故事

### US-01 首次打开 App

**作为** 第一次使用该 App 的初学者，  
**我希望** 打开 App 后能直接看到今天要练什么，  
**以便** 不用思考该练什么，立刻开始练习。

**验收标准**：
- [ ] 启动后默认进入"今日练习"页；
- [ ] 显示当天日期与 Day N 编号（1-7）；
- [ ] 展示当天练习任务卡片列表（≥ 2 项）；
- [ ] 无需注册 / 登录。

**优先级**：P0

---

### US-02 查看今日练习

**作为** 当天有练习计划的用户，  
**我希望** 能清楚看到每项练习的目标和时长，  
**以便** 合理安排练习顺序。

**验收标准**：
- [ ] 每个任务卡片包含：任务名、目标、预计时长；
- [ ] 已完成的任务有视觉标记；
- [ ] 点击任务可进入对应功能页；
- [ ] 总预计时长 ≤ 25 分钟。

**优先级**：P0

---

### US-03 进入调音器

**作为** 即将开始练习的用户，  
**我希望** 在首页或练习页能快速进入调音器，  
**以便** 练习前先确认琴弦音准。

**验收标准**：
- [ ] 首页提供调音器入口；
- [ ] 进入调音器时清晰提示"开始需要麦克风权限"；
- [ ] 首次使用调音器时弹出系统权限申请；
- [ ] 不在 App 启动时申请麦克风权限。

**优先级**：P0

---

### US-04 调 G 弦

**作为** 需要调 G 弦的用户，  
**我希望** 手动选择 G 弦后能看到当前频率与偏差提示，  
**以便** 把 G 弦调到标准音。

**验收标准**：
- [ ] 提供 G / C / E / A 四个弦的切换按钮；
- [ ] 显示当前频率（Hz）；
- [ ] 显示目标频率（G = 392.00 Hz）；
- [ ] 给出"偏高 / 偏低 / 接近准确"三种状态；
- [ ] 接近准确阈值：±10 cents（体验目标）。

**优先级**：P0

---

### US-05 调 C 弦

**作为** 需要调 C 弦的用户，  
**我希望** 手动选择 C 弦后能看到当前频率与偏差提示，  
**以便** 把 C 弦调到标准音（C = 261.63 Hz）。

**验收标准**：
- [ ] 弦切换响应 < 500ms；
- [ ] 偏差提示与 G 弦一致。

**优先级**：P0

---

### US-06 调 E 弦

**作为** 需要调 E 弦的用户，  
**我希望** 手动选择 E 弦后能看到当前频率与偏差提示，  
**以便** 把 E 弦调到标准音（E = 329.63 Hz）。

**验收标准**：同 US-04。

**优先级**：P0

---

### US-07 调 A 弦

**作为** 需要调 A 弦的用户，  
**我希望** 手动选择 A 弦后能看到当前频率与偏差提示，  
**以便** 把 A 弦调到标准音（A = 440.00 Hz）。

**验收标准**：同 US-04。

**优先级**：P0

---

### US-08 查看和弦指法图

**作为** 学习新和弦的用户，  
**我希望** 能浏览和弦库并查看指法图，  
**以便** 学习按弦方法。

**验收标准**：
- [ ] 和弦库至少包含 C / Am / F / G / G7 / Dm / Em；
- [ ] 每个和弦有指法图（自绘静态图）；
- [ ] 显示和弦名称、组成音、推荐指位；
- [ ] 无 TAB / 五线谱 / 简谱。

**优先级**：P0

---

### US-09 开始节拍器

**作为** 需要稳定节拍感的用户，  
**我希望** 设置 BPM 后能开始节拍器，  
**以便** 配合练习单音或和弦转换。

**验收标准**：
- [ ] BPM 范围 50 - 200，默认 80；

## 9. 第一版 7 天练习计划

MVP 预设 7 天循环练习计划。完成 Day 7 后回到 Day 1。

> **轮换逻辑**：`Day N = (自安装起的天数 % 7) + 1`，Day 1-7 严格循环。例如：
> - 安装当天 = Day 1；
> - 安装后第 7 天 = Day 1；
> - 安装后第 8 天 = Day 2。
>
> MVP 不做个性化、不做 AI 推荐、未完成任务**不自动惩罚**。

| Day | 主题 | 任务列表 | 预计总时长 |
|-----|------|----------|------------|
| Day 1 | 认识琴弦 | 调音 G/C/E/A → 单音 C / E 练习 → 节拍器 80 BPM 跟拍 | 15 分钟 |
| Day 2 | 单音进阶 | 调音 → 单音 C / D / E / F / G 练习 → 节拍器 80 BPM | 15 分钟 |
| Day 3 | 第一个和弦 | 调音 → 学习 C 和弦 → 节拍器 80 BPM → 录音 1 段 → 自评 | 18 分钟 |
| Day 4 | 和弦转换 | 调音 → C ↔ Am 转换 → 节拍器 80 BPM → 录音 1 段 → 自评 | 18 分钟 |
| Day 5 | 更多和弦 | 调音 → F / G 和弦指法 → 节拍器 90 BPM → 录音 1 段 → 自评 | 20 分钟 |
| Day 6 | 综合练习 | 调音 → C - Am - F - G 循环 → 节拍器 100 BPM → 录音 1 段 → 自评 | 20 分钟 |
| Day 7 | 复习巩固 | 调音 → 用户任选和弦 / 单音组合 → 节拍器 90 BPM → 录音 1 段 → 自评 | 20 分钟 |

**任务内容范围（统一适用所有 Day）**：
- 单音练习：基础音名按弦（C / D / E / F / G / A / B）；
- 和弦：C / Am / F / G / G7 / Dm / Em（Day 1-2 不涉及和弦，Day 3 起逐步引入）；
- 节拍器：4/4 拍，BPM 在 50-200 内可调，默认 80；
- 录音：≤ 5 分钟，本地保存；
- 自评：好 / 一般 / 需改进。

**一次练习的 `practice_type` 字段**：因一次练习可包含多个活动（调音 + 单音 + 节拍 + 录音），`practice_type` 字段存储为**逗号分隔字符串**，如 `"tuner, single_note, recording"` 或 `"chord, metronome, recording"`。完整枚举值：`tuner` / `single_note` / `chord` / `metronome` / `recording` / `self_assessment` / `mixed`。

**完成状态**：
- 每项任务可单独标记"完成"；
- 完成状态仅本地保存，不强制全部完成；
- 未完成不扣分、不惩罚、不影响第二天内容。

每条练习记录（`practice_records` 表）至少包含：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | String / UUID | ✅ | 记录 ID，主键 |

| `practice_type` | String | ✅ | 练习类型，逗号分隔（如 `"tuner,chord,recording"`） |
| `practice_content` | String | ✅ | 练习内容描述（如 `"Day 4: C ↔ Am 转换"`） |
| `duration_seconds` | int | ✅ | 练习时长（秒） |
| `is_completed` | bool | ✅ | 是否完成 |
| `self_assessment` | String \| null | ❌ | 自评结果：`good` / `neutral` / `needs_improvement` / `null` |
| `audio_file_path` | String \| null | ❌ | 录音文件路径（相对 App 私有目录） |
| `created_at` | DateTime | ✅ | 创建时间 |
| `updated_at` | DateTime | ✅ | 最后修改时间 |

### 10.2 其他本地数据

| 数据 | 存储 | 说明 |
|------|------|------|
| 用户设置 | SharedPreferences | 默认 BPM（默认 80）、节拍器音量 |
| 录音文件 | App 私有目录（`getApplicationDocumentsDirectory()`） | 命名：`<timestamp>_<uuid>.m4a` |
| 7 天计划配置 | 内置 Dart 代码常量 | 不放在数据库（便于后续调整） |
| 频率参考表 | 内置 Dart 代码常量 | 不放在数据库 |
| 和弦指法图 | 内置 PNG / SVG 资源（`assets/chords/`） | 不放在数据库 |

### 10.3 数据生命周期

- 练习记录：永久本地保存（用户可单条删除）；
- 录音文件：与关联练习记录同步删除（删除记录时一并删除文件）；
- 用户设置：随 App 卸载消失；
- 录音文件被外部删除：UI 隐藏回放按钮，记录本身保留，`audio_file_path` 字段保留原值。

### 10.4 数据不上传原则

- 无任何后端服务；
- 无任何联网 API；
- 无任何第三方分析 SDK；
- 卸载 App = 数据全部清除；
- **不申请** Android 媒体权限（`READ_MEDIA_AUDIO` 等），因为 MVP 不写公共目录，避免 Android 13+ 媒体权限复杂度。

### 11.2 Android 版本

| 项目 | 规格 |
|------|------|
| `minSdk` | **23（Android 6.0）** |
| `targetSdk` | 34（推荐 35） |
| `compileSdk` | ≥ 33（permission_handler 3.1+ 要求） |

**minSdk 决策原因**：

- T001 研究发现 `record` 7.x 要求 Android minSdk 23，与早期 `minSdk 21` 候选存在冲突；
- 优先保证录音、PCM 音频流、调音器等音频能力稳定，**暂定 minSdk 23**；
- Android 6.0 以下用户占比极少（具体数字需 T001B 复核），可接受损失；
- 如 T005 实际验证发现依赖支持更低版本，由 Chief Architect 再评估是否降低。

### 11.3 技术栈参考

> 详细技术选型由 T003_FINALIZE_TECH_ARCHITECTURE 决策，本节仅列出 PRD 视角的非可议项。

- Flutter + Dart；
- 本地 SQLite（Drift + sqlite3_flutter_libs）；
- 无后端服务；
- 无 AI / API 集成；
- 麦克风权限延迟申请（首次进入调音器/录音时）；
- 录音本地保存，不写公共目录（避免 Android 13+ 媒体权限复杂度）。

### 11.4 权限

| 权限 | 必选 | 用途 |
|------|------|------|
| `android.permission.RECORD_AUDIO` | ✅ | 调音器、录音 |
| `android.permission.MODIFY_AUDIO_SETTINGS` | 可选 | 蓝牙耳机支持 |
| `android.permission.WRITE_EXTERNAL_STORAGE` | ❌ | MVP 不写公共目录 |
| `android.permission.READ_MEDIA_AUDIO` | ❌ | MVP 不读公共目录 |
| `android.permission.INTERNET` | ❌ | MVP 无任何联网 |

**申请时机**：用户首次点击"开始调音"或"开始录音"时申请，**不在 App 启动时申请**。

### 11.5 未来 iOS 阶段必须补的内容（预留参考）

### 15.2 体验验收

- [ ] 用户能在 3 分钟内完成首次调音 → 练习 → 录音 → 保存全流程；
- [ ] 单次练习时长 ≤ 25 分钟；
- [ ] 界面清晰，无歧义；
- [ ] 错误提示友好（不崩溃，有重试入口）；
- [ ] 调音器权限申请延迟到用户首次进入调音器时；
- [ ] 录音权限申请延迟到用户首次录音时。

### 15.3 范围守卫（不通过验收的反例）

如出现以下任一情况，**视为 MVP 不通过**：

- 内置任何商业歌曲完整歌词 / 曲谱 / 音频；
- 出现任何联网请求（包括但不限于 API、统计、广告）；
- 引入任何第三方分析 SDK；
- minSdkVersion < 23；
- T004 创建了 ios/ 目录；
- 实现 §7 列出的任何"不做什么"功能；
- 调音器承诺"专业级精度"文案；
- 录音文件存储到 App 私有目录以外的位置；
- 录音文件可被分享或导出；
- 练习记录同步到任何云端；
- 启用 AI 自动评分（任何形式）；
- 提供用户输入歌曲名 / 歌词 / 链接的入口。
