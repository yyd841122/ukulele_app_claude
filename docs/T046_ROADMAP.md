## 1. P1 Foundation — 补齐 v1.1.0 承诺 + PCM 流暴露

**目标**：完成 v1.1.0 PRD 承诺但未落地的能力；暴露 T031E PCM 实时流（**P2 硬前置**）。

**包含任务**（3 任务可见节奏）：
- P1-T1：**T031E 扩展 PCM `Stream<Float32>`**（关键前置，决定 P2 可行性）
- P1-T2：和弦库 + 单音库补齐（+ G7 / Dm / Em / B）+ 调音器真实时化 + 节拍器可听音
- P1-T3：设置页 UI 暴露 + 深色 token

**3 任务 demo 脚本**：
1. 装 APK → 进首页 → 进入"和弦库" → 看到 7 个和弦（C / Am / F / G / G7 / Dm / Em）。
2. 同 APK → 进入"单音练习" → 看到 7 个音名（C / D / E / F / G / A / B）。
3. 同 APK → 进入"调音器" → 弹 G 弦 → 看到 cents 在 ±10 内实时跳动。
4. 同 APK → 进入"节拍器" → 听到 80 BPM 4/4 拍可听点击音。
5. 同 APK → 设置页 → 看到默认 BPM 滑块 + 节拍器音量滑块。

**真机可见成果**：5 项真机可见。

**关闭条件**：
- T031E PCM 流暴露 + 4 项 §PRD_v2 §7.2 P0 100% 通过
- T010 调音器精度 spike 报告归档
- 单元测试 + widget test 通过

**前置**：v1.1.0 真实音频 MVP（b2983fa）已发布。

## 2. P2 Interactive Slice — 9 步闭环第一课

**目标**：交付"教学说明 → 倒计时 → 时间轴谱 → 当前拍高亮 → 用户弹奏 → 起音/节奏检测 → 客观反馈 → 录音回放 → 课程完成"端到端循环。

**包含任务**（3 任务可见节奏）：
- P2-T1：节拍器时钟同步 + 时间轴谱组件 + 倒计时组件 + 当前拍高亮（UI 层）
- P2-T2：起音检测（能量包络 + spectral flux）+ 节奏对齐检测（80 BPM 网格）+ 客观反馈 UI
- P2-T3：录音流接入 + 课程完成状态机 + **T041 旧路径退役** + widget test

**3 任务 demo 脚本**：
1. 装 APK → 进首页 → Day 4 任务卡 → 看到"开始课程"入口。
2. 同 APK → 进入 Lesson → 看到教学说明 → 听到 4 拍倒计时 → 看到时间轴谱 + 当前拍高亮 → 60 BPM 热身 → 80 BPM 目标 → 弹奏 → 看到"24/32 拍对齐"客观反馈 → 录音回放 → 课程通过 toast。
3. 同 APK → Day 5 任务卡 → 仍冻结（保持 T041 静态叠加，不扩展课程数量）。

**真机可见成果**：1 条完整 9 步路径。

**关闭条件**：
- 9 步闭环 widget test + 真机演示录屏
- Day 3/5/6/7 课程数量**仍冻结**（真机验证后由 P4 启动）
- T041 的"自评=好 → 关卡通过 toast"路径**退役**
- 客观反馈：24/32 对齐 + ≤2 次闷音切换 → 通过

**前置**：P1 完成（含 T031E PCM 流暴露）。

**关键不变量**：
- 不修改 v1.1.0 已发布的 7 天计划常量。
- 不修改 `MetronomeSetting` / 不存储节奏型。
- 不修改 Drift schema（schemaVersion 仍为 2）。
- 不引入新依赖。
- 不申请新权限。
- 不引入 INTERNET。

## 3. P3 Audio Intelligence — 调音器精度 + 音高评分 + 和弦识别

**目标**：在 P2 互动闭环基础上，引入本地音高评分与和弦识别能力。

**包含任务**（3 任务可见节奏）：
- P3-T1：调音器精度提升（autocorrelation / YIN 选择）
- P3-T2：单音音高评分服务
- P3-T3：和弦识别（chroma 模板匹配）

**3 任务 demo 脚本**：
1. 装 APK → 进调音器 → 弹 G 弦 → 看到 cents + 音高评分。
2. 同 APK → 进单音练习 → 弹 C → 看到评分。
3. 同 APK → 切到无网络 → 评分仍工作。

**真机可见成果**：调音器精度提升 + 单音评分 + 和弦识别。

**关闭条件**：
- ±20 cents 准确率 ≥ 80% on-device
- 和弦识别准确率 ≥ 70% on-device（5 个常见和弦）
- 完全离线（关闭网络后仍工作）
- 无 INTERNET 权限

**前置**：P2 完成。

## 4. P4 Curriculum — 课程地图 + 进度 + 复习 + 原创歌曲

**目标**：建立 5 阶段课程地图、原创歌曲库与复习推荐层。

**包含任务**（3 任务可见节奏）：
- P4-T1：Day 3/5/6/7 课程扩展（P2 关闭门解除后启动）+ 动态曲谱引擎
- P4-T2：原创歌曲 ≥ 8 首（CC0，05-music-domain-expert 出具）+ 节奏型扩展（静态 SVG）
- P4-T3：复习推荐（基于薄弱点和弦 + 薄弱转换）+ 课程地图 UI

**3 任务 demo 脚本**：
1. 装 APK → 进课程地图 → 看到 5 阶段（单音 → 和弦 → 节奏 → 弹唱片段 → 完整曲目）。
2. 同 APK → 选 1 首原创歌曲 → 跟弹 → 看到动态光标。
3. 同 APK → 切到无网络 → 跟弹仍工作。

**真机可见成果**：5 阶段课程地图 + 跟弹 1 首自制曲目。

**关闭条件**：
- ≥8 首原创歌曲（CC0） + 版权审查通过
- 复习推荐点击率 ≥ 20%
- 完全离线
- 跟弹节奏评分每小节 pass/fail 准确率 ≥ 80%

**前置**：P3 完成 + P2 关闭门解除。

## 8. 阶段交付节奏铁律

> 任意连续 3 个开发任务必须产生真机可见成果（APK 可装、用户能感知）。
> 每个阶段必须包含 1 个"真机 demo 录屏"环节（Phase Gate）。
> 阶段关闭前必须做"v1.1.0 兼容性 + v2 能力增长"双口径验收。
> **P2 关闭 = 关键门**：Day 3/5/6/7 课程数量冻结到 P2 真机验证前。
