## 1. P1 — 补齐 v1.1.0 承诺

**目标**：完成 v1.1.0 PRD 承诺但未落地的能力；建立 v2 的稳定基线。

**包含任务**（3 任务可见节奏）：
- P1-T1：和弦库 + 单音库补齐（+ G7 / Dm / Em / B）
- P1-T2：调音器真实时化（PCM + autocorrelation / YIN）+ 节拍器可听音
- P1-T3：设置页 UI 暴露 + 深色 token

**3 任务 demo 脚本**：
1. 装 APK → 进首页 → 进入"和弦库" → 看到 7 个和弦（C / Am / F / G / G7 / Dm / Em）。
2. 同 APK → 进入"单音练习" → 看到 7 个音名（C / D / E / F / G / A / B）。
3. 同 APK → 进入"调音器" → 弹 G 弦 → 看到 cents 在 ±10 内实时跳动。
4. 同 APK → 进入"节拍器" → 听到 80 BPM 4/4 拍可听点击音。
5. 同 APK → 设置页 → 看到默认 BPM 滑块 + 节拍器音量滑块。

**真机可见成果**：5 项真机可见。

**关闭条件**：
- 4 项 §4.3 P0 100% 通过
- T010 调音器精度 spike 报告归档
- 单元测试 + widget test 通过

**前置**：v1.1.0 真实音频 MVP（b2983fa）已发布。

## 2. P2 — 第一垂直切片 + 关卡化

**目标**：交付"教学 → 静态练习谱 → 跟弹 → 基础反馈 → 录音复盘 → 关卡通过"端到端循环。

**包含任务**（3 任务可见节奏）：
- P2-T1：Lesson 模型扩展至 Day 3/5/6/7（沿用 T041 模板）
- P2-T2：Lesson page + 关卡通过 toast
- P2-T3：widget test 覆盖 4 个 Lesson

**3 任务 demo 脚本**：
1. 装 APK → 进首页 → Day 4 任务卡 → 看到"开始课程"入口。
2. 同 APK → 进入 Lesson → 60 BPM 热身 → 80 BPM 目标 → 录音 30 秒 → 自评 → "关卡通过"toast。
3. 同 APK → Day 5 任务卡 → 同样的 Lesson 模板（扩展后）。

**真机可见成果**：1 条完整路径（Day 4）+ 1 条扩展路径（Day 5）。

**关闭条件**：
- 4 个 Lesson widget test 通过
- 关卡通过 toast 真机演示录屏
- 4 个 Lesson 内容由 05-music-domain-expert 出具并通过 08-compliance-reviewer 版权审查

**前置**：P1 完成。

**关键不变量**：
- 不修改 v1.1.0 已发布的 7 天计划常量。
- 不修改 `MetronomeSetting` / 不存储节奏型。
- 不修改 Drift schema（schemaVersion 仍为 2）。
- 不引入新依赖。
- 不申请新权限。

## 3. P3 — 本地音高识别

**目标**：在 P1 调音器真实时化基础上，引入本地音高评分能力。

**包含任务**（3 任务可见节奏）：
- P3-T1：录音器 PCM 实时流暴露
- P3-T2：本地音高评分服务（autocorrelation / YIN / MPM 选一）
- P3-T3：调音器 UI 集成评分

**3 任务 demo 脚本**：
1. 装 APK → 进调音器 → 弹 G 弦 → 看到 cents + 评分。
2. 同 APK → 进单音练习 → 弹 C → 看到评分。
3. 同 APK → 切到无网络 → 评分仍工作。

**真机可见成果**：调音器精度提升 + 单音评分雏形。

**关闭条件**：
- ±20 cents 准确率 ≥ 80% on-device
- 完全离线（关闭网络后仍工作）
- 无 INTERNET 权限

**前置**：P2 完成。

## 4. P4 — 统计与复习

**目标**：建立学习记录的统计与复习推荐层。

**包含任务**（3 任务可见节奏）：
- P4-T1：Drift schemaVersion 3 迁移（+ lessonId / accuracyScore / bpmUsed）
- P4-T2：首页统计 tile（streak / 周累计 / 完成率）
- P4-T3：复习推荐（基于薄弱点和弦 + 薄弱转换）

**3 任务 demo 脚本**：
1. 装 APK → 进首页 → 看到 streak=3 + 周累计 80 分钟 + 完成率 75%。
2. 同 APK → 进"今日推荐" → 看到"昨日 Am→C 评分 < 60，今日推荐 Am→C 转换练习"。
3. 同 APK → 切到无网络 → 统计仍工作。

**真机可见成果**：首页统计 + 复习推荐。

**关闭条件**：
- schemaVersion 3 迁移 + 回放测试通过
- 推荐命中 ≥ 20%（点击率）
- 完全离线

**前置**：P3 完成。

## 5. P5 — 动态曲谱 + 跟弹 + 节奏评分

**目标**：交付 v2 最有差异化主张的能力——跟弹 + 实时光标 + 节奏评分。

**包含任务**（3 任务可见节奏）：
- P5-T1：动态曲谱资源（自制曲目 SVG + 章节标记）
- P5-T2：跟弹模式（实时光标 + 录音流）
- P5-T3：节奏评分（onset detection + 每小节 pass/fail）

**3 任务 demo 脚本**：
1. 装 APK → 进曲目库 → 选 1 首自制曲目 → 看到动态光标。
2. 同 APK → 跟弹 → 看到每小节 pass/fail 实时反馈。
3. 同 APK → 切到无网络 → 跟弹仍工作。

**真机可见成果**：跟弹 1 首自制曲目（≤ 8 小节）。

**关闭条件**：
- 跟弹 1 首自制曲目 on-device 演示
- 节奏评分每小节 pass/fail 准确率 ≥ 80%
- 完全离线

**前置**：P4 完成。

## 6. P6 — 账号 / 云同步 / 订阅

**目标**：在合规前置全部完成的前提下，引入账号 / 云同步 / 订阅。

**包含任务**（3 任务可见节奏）：

- P6-T2：订阅墙 + 高级统计
- P6-T3：iOS 适配（如启动）

**3 任务 demo 脚本**：
1. 装 APK → 注册 → 登录。
2. 同 APK → 删本地数据 → 重装 → 恢复记录。
3. 同 APK → 看到订阅墙。

**真机可见成果**：账号 + 同步 + 订阅。

**关闭条件**：
- 全部 §10 合规前置完成

- DMCA / GDPR / CCPA / COPPA 评估归档

**前置**：P5 完成 + 合规前置。

## 8. 阶段交付节奏铁律

> 任意连续 3 个开发任务必须产生真机可见成果（APK 可装、用户能感知）。
> 每个阶段必须包含 1 个"真机 demo 录屏"环节（Phase Gate）。
> 阶段关闭前必须做"v1.1.0 兼容性 + v2 能力增长"双口径验收。

## 9. 风险与依赖

| 风险 | 影响 | 缓解 |
|------|------|------|
| P1 调音器精度不达 ±10 cents | P1 阻塞 | T009 spike 报告归档后启动 P1 |
| P3 本地算法准确度不足 | P3 阻塞 | 离线可选阈值（如 ±20 cents） |
| P4 schemaVersion 3 迁移失败 | v1.1.0 数据丢失 | 兼容回放 + 强制备份提示 |
| P5 跟弹评分准确度不足 | P5 阻塞 | 评分可关闭，不阻塞关卡通过 |
| P6 合规前置未完成 | 商业化延期 | 从 P4 起每月 readiness 评审 |
| 用户对 6 阶段路线失去耐心 | 用户流失 | P2 完成即发布 v2.0.0 预览 |
