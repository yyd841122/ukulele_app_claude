# Handoff 模板

> 当一个 Agent 完成一个任务，需要交接给下一个 Agent 或向 Chief Architect 报告时使用此模板。

---

## Handoff 文档模板

```markdown
# Handoff: [任务名称]

## 基本信息

| 项目 | 内容 |
|------|------|
| Task ID | [Task ID] |
| 执行 Agent | [Agent 编号和名称] |
| 交接给 | [下一个 Agent 或 N/A] |
| 日期 | [YYYY-MM-DD] |
| 状态 | [进行中/已完成/阻塞] |

## 输入上下文

[描述这个任务接收到的输入]

**输入来源**：
- [来源 1]
- [来源 2]

## 已完成内容

### 文件创建

- [文件 1]
- [文件 2]

### 文件修改

- [文件 1]

### 功能实现

- [功能 1]
- [功能 2]

## 验证结果

[描述如何验证这些已完成的内容]

### 验证方式

- [ ] 验证项 1
- [ ] 验证项 2

### 验证结果

- [ ] PASS / FAIL
- [ ] PASS / FAIL

## 未完成内容

[描述哪些内容还未完成]

### 原因

[解释为什么未完成]

### 阻塞因素

- [阻塞因素 1]
- [阻塞因素 2]

## 风险点

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| [风险 1] | [影响] | [措施] |
| [风险 2] | [影响] | [措施] |

## 下一步建议

### 立即下一步

1. [步骤 1]
2. [步骤 2]

### 后续任务

- [后续任务 1]
- [后续任务 2]

## 备注

[其他需要说明的事项]

---

## 使用说明

1. 当 Agent 完成一个任务或需要交接时，填写此模板
2. 将模板保存到 `handoff/[task-id]-handoff.md`
3. 将副本发送给下一个 Agent 和 Chief Architect
4. Chief Architect 审核后，更新任务状态

---

## 文档版本

| 版本 | 日期 | 修改内容 |
|------|------|----------|
| 0.1 | 2026-06-19 | 初始版本 |
```

---

## Handoff 流程图

```
Agent 执行任务
    ↓
任务完成
    ↓
填写 Handoff 文档
    ↓
提交给 Chief Architect
    ↓
Chief Architect 审核
    ↓
通过
    ↓
交接给下一个 Agent / 任务完成
```
