## Key Product Patterns

总结三类竞品中的共同产品模式：

1. **"今日练什么"是产品最关键的一屏**
   - AI 学园：每日关卡、每日任务
   - 教学类：当前学习路径上的下一节
   - 调音器类：通常没有"今日"，但有"上次状态"
   - **结论**：MVP 的"首页 = 今日练习"是经过验证的产品模式

2. **"练习 → 立即听效果"是核心闭环**
   - 录音 + 回放几乎是标配
   - 自评可以是 AI，也可以是手动
   - **结论**：MVP 录音 + 回放 + 手动自评形成最简闭环

3. **调音器是黏性入口**
   - 几乎所有弦乐类 App 都有调音器
   - **结论**：MVP 调音器是必须，且要做得"打开就能用"

4. **节拍器是基本功训练器**
   - 节拍器 50-200 BPM 是通用范围
   - 多种节奏型在专业节拍器中是标配，MVP 只做基础 4/4 即可
   - **结论**：MVP 节拍器 = BPM 调节 + 基础 4/4 拍

5. **和弦库是查阅工具**
   - 通常以静态图 / 简单动图展示
   - MVP 用 SVG / 静态 PNG 即可
   - **结论**：MVP 不做交互式指法动画

## Features Worth Borrowing

适合 MVP 借鉴的功能：

| 功能 | 借鉴来源 | MVP 实施方式 |
|------|----------|--------------|
| 首页"今日练习"卡片 | AI 学园 | 简单列表 + 状态标记 |
| 内置调音器（手动选弦） | 调音器类 | G/C/E/A 四弦切换 |
| 内置节拍器 | 调音器类 | 50-200 BPM 滑块 |
| 录音后立即回放 | 教学类 | record + just_audio |
| 三档手动自评 | 调音器类 | 好 / 一般 / 需改进 |
| 练习历史日历 | 打卡类 | 简单列表 + 日期分组 |
| 静态和弦指法图 | 教学类 | 内置 PNG / SVG 资源 |

## Features to Avoid in MVP

MVP **明确不要做**的功能：

| 功能 | 为什么不 MVP 做 |
|------|-----------------|
| AI 自动音高评分 | 算法成本高，超出 MVP 范围 |
| AI 节奏分析 | 同上 |
| AI 和弦识别 | 同上 |
| AI 曲谱生成 | 版权与算力双重风险 |
| 视频课程 | 内容成本极高 |
| 完整歌曲库 | 版权高危区 |
| 社区 / 排行榜 / 好友 | MVP 范围内不需要 |
| 云同步 / 账号系统 | MVP 离线优先 |
| 内购 / 订阅 | MVP 不商业化 |
| 广告 | MVP 体验优先 |
| iOS / Web / 桌面 | MVP Android only |
| 多乐器 | MVP 单乐器（尤克里里） |
| Low G / 异国调弦 | MVP 标准 GCEA |
| TAB / 五线谱 / 简谱 | MVP 范围外 |
| 曲谱编辑器 | 版权与复杂度双重风险 |
| 录音导出 / 分享 | 版权与隐私双重风险 |
| 推送通知 | MVP 简单化 |

## MVP Implications

对 MVP 的直接影响：

1. **MVP 不做 AI**：这与 MVP_SCOPE §2.2 一致。调音器是单帧频率检测，不是 AI 评分。
2. **MVP 做"听 + 看 + 评"最小闭环**：录音 → 回放 → 手动三档自评 → 保存记录。
3. **MVP 调音器只做基础方向提示**：不承诺专业级精度。
4. **MVP 内置内容必须可控**：
   - 和弦图 = 通用知识（无版权风险）
   - 单音练习 = 音名 + 频率，通用知识
   - 练习曲 = 公共领域 / 自制
   - 完整歌曲 = **不做**
5. **MVP "今日练习"是简单预设列表**：不需要 AI 推荐算法，按周期轮换。

## Open Questions for T002

需要 T002 阶段解决的产品问题：

1. **"今日练习"的内容如何轮换**？
   - 按天序号轮换（Day 1/2/3...）？
   - 按周轮换（Week 1/2...）？
   - 还是按用户状态智能推荐（暂不推荐，MVP 简单化）？
2. **调音器是否需要"自动选弦"**？MVP 限定手动选弦，自动选弦是 V1 增强。
3. **节拍器是否需要节奏型**？MVP 限定 4/4 基础拍，其他节奏型后期。
4. **录音文件保存多久**？MVP 暂定永久本地保存，文件大小需要规划。
5. **自评如何与录音绑定**？MVP 一次练习 = 一条录音 + 一条记录 + 一个自评，是否需要"一次练习多段录音"？MVP 限定一对一。
6. **是否需要"导出/分享录音"按钮**？MVP 不做（合规风险）。
7. **iOS 暂留接口**：iOS 上架时是否复用一套 UI 资源？待 T002 决定。
